# Story 2.10: SSH Connection Failure

**Epic:** Epic 2 - deploy.sh Core Functionality Testing
**Priority:** Must-have

## Description

Verify clear error when VPS is unreachable during deploy.

## Acceptance Criteria

- [x] Before rsync, SSH connection test performed
- [x] On failure: error "Cannot connect to ${VPS_USER}@${VPS_IP}:${SSH_PORT}"
- [x] Suggests checking SSH key, user, IP, port
- [x] Exit code 1, no deploy attempted

## Dependencies

Story 2.4, 2.5

## Testing Notes

```bash
cat > .env.deploy << 'EOF'
VPS_USER=deploy
VPS_IP=255.255.255.255
DOMAIN=example.com
DOMAIN_TYPE=static
EOF

./deploy.sh
echo $?  # Should be 1
```
