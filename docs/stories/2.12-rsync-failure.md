# Story 2.12: Rsync Failure

**Epic:** Epic 2 - deploy.sh Core Functionality Testing
**Priority:** Must-have

## Description

Verify script handles rsync failures (network, permissions, disk space).

## Acceptance Criteria

- [ ] If rsync exits with non-zero, script exits with error
- [ ] Error message includes rsync output
- [ ] Common causes suggested: network issue, disk full, permission denied
- [ ] Exit code reflects rsync failure

## Dependencies

Stories 2.4, 2.5

## Testing Notes

```bash
# Test with invalid VPS IP or blocked port
cat > .env.deploy << 'EOF'
VPS_USER=deploy
VPS_IP=10.255.255.1
DOMAIN=example.com
DOMAIN_TYPE=static
EOF

./deploy.sh
echo $?  # Should be non-zero (rsync failure)
```
