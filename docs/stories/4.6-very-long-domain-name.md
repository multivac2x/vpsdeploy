# Story 4.6: Very Long Domain Name

**Epic:** Epic 4 - Error Handling and Edge Cases
**Priority:** Could-have

## Description

Verify handling of unusually long domain names (63 character limit per label).

## Acceptance Criteria

- [x] Domain with 63-character subdomain label
- [x] Registry stores full domain correctly
- [x] Caddyfile generated without line wrapping issues
- [x] Directory paths created with long name
- [x] No truncation in logs or status displays

## Dependencies

Story 1.10

## Testing Notes

```bash
# Use a domain with 63-character subdomain label
# e.g., aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa.example.com

cat > .env.deploy << 'EOF'
VPS_USER=deploy
VPS_IP=your.vps.ip
DOMAIN=aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa.example.com
DOMAIN_TYPE=static
EOF

./setup-vps.sh --dry-run --verbose

# Check that full domain appears in output without truncation
```
