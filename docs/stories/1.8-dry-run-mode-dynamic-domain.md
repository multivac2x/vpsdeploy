# Story 1.8: Dry-Run Mode - Dynamic Domain

**Epic:** Epic 1 - setup-vps.sh Core Functionality Testing
**Priority:** Must-have

## Description

Verify dry-run mode includes Node.js/PM2 installation and setup for dynamic domains.

## Acceptance Criteria

- [x] `./setup-vps.sh --dry-run --verbose` with `DOMAIN_TYPE=dynamic` shows Node.js/PM2 installation checks
- [x] Shows PM2 setup phase
- [x] Shows port conflict check
- [x] No actual changes made, exit code 0 if validation passes

## Dependencies

Stories 1.1-1.6

## Testing Notes

```bash
cat > .env.deploy << 'EOF'
VPS_USER=deploy
VPS_IP=your.vps.ip
DOMAIN=test-dynamic.com
DOMAIN_TYPE=dynamic
DOMAIN_PORT=3000
EOF

./setup-vps.sh --dry-run --verbose > output.txt
echo $?  # Should be 0

# Verify dynamic-specific content
grep -E "Node.js|PM2|port.*3000" output.txt
```
